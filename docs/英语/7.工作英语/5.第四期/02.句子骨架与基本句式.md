# 第四期 - 02. 句子骨架与基本句式

## 一、三大核心骨架

任何复杂的英文长难句，都可以拆解为以下三种基础核心骨架之一：

### 核心骨架 1：主谓 (Subject + Verb)
主要用于动作由主角自己发出，不需要作用于其他客体（使用不及物动词）。
*   **实例**：The server crashed. (服务器崩溃了。)
    *   **主语 (S)**：The server (服务器，发出者)
    *   **谓语 (V)**：crashed (崩溃，不及物动词过去式)
    *   **结构解析**：主语 + 谓语（不需要跟任何宾语，句意已完整）。

### 核心骨架 2：主谓宾 (Subject + Verb + Object)
最常用的结构。主角发出一个“动作”，作用到一个“对象（客体）”上（使用及物动词）。
*   **实例 1**：I write code with C#.
    *   **中文翻译**：我用 C# 写代码。
    *   **结构解析**：I (主语) + write (谓语) + code (宾语) + [with C#] (方式状语)。
*   **实例 2**：He fixed a bug with Vue3 yesterday.
    *   **中文翻译**：他昨天用 Vue3 修复了一个 bug。
    *   **结构解析**：He (主语) + fixed (谓语，过去式) + a bug (宾语) + [with Vue3] (方式状语) + [yesterday] (时间状语)。

### 核心骨架 3：主系表 (Subject + Linking Verb + Predicative)
用于说明主语“是什么”或“处于什么状态”。系动词相当于赋值符号（`=`）。
*   **实例 1**：The UI component is green. (UI 组件是绿色的。)
    *   **赋值公式**：`The UI component = green`
    *   **结构解析**：The UI component (主语) + is (系动词) + green (形容词作表语)。
*   **实例 2**：The API is ready. (API 准备好了。)
    *   **结构解析**：The API (主语) + is (系动词) + ready (形容词作表语)。

## 二、陈述句（肯定/否定）

陈述句用于陈述事实或状态：
*   **肯定句**：The code works. (代码能跑。)
    *   **结构解析**：The code (主语) + works (不及物动词作谓语，单三)。
*   **否定句**：The code does not work. (代码不能跑。)
    *   **结构解析**：借助否定助动词 `does not`，后面的实义动词恢复原形 `work`。

## 三、一般疑问句 (Yes/No Questions)

用 **Yes** 或 **No** 回答的疑问句，其构造方法取决于句中是否存在 Be 动词：

### 1. 句中含有 Be 动词 (is/am/are/was/were)
直接把 Be 动词挪到最前面，其余按原顺序排列。
*   陈述句：The page **is** slow.
*   疑问句：**Is** the page slow? ❓
    *   **结构解析**：Is (系动词前置) + the page (主语) + slow (表语)？

### 2. 句中只有普通实义动词
必须请助动词（`do` / `does` / `did`）来句首帮忙，后面的实义动词**恢复原形**。
*   陈述句：The API works.
*   疑问句：**Does** the API **work**? ❓
    *   **结构解析**：Does (单三助动词) + the API (主语) + work (动词原形作谓语)？
*   否定提问：**Doesn't** the API work? (这个接口不能工作吗？)

## 四、特殊疑问句 (Wh- Questions)

针对具体时间、地点、原因、方式等信息进行提问，需要使用特殊疑问词（`Who`, `What`, `Where`, `When`, `Why`, `How`）。

*   **构造规律**：**特殊疑问词 + 一般疑问句语序**。
*   **注意点**：实义动词同样需要**恢复原形**。
*   **演变流程**：
    1.  陈述句：*You fixed the bug.* (你修复了这个 bug。)
    2.  一般疑问句：*Did you fix the bug?* (你修复了这个 bug 吗？)
    3.  特殊疑问句：**How** did you fix the bug? ❓ (你是怎么修复这个 bug 的？)
        *   **结构解析**：How (方式疑问副词) + did (助动词) + you (主语) + fix (谓语动词原形) + the bug (宾语)？

---

## 五、反义疑问句 (Tag Questions)

在陈述句末尾加上一个“小尾巴”，用于向对方确认信息。遵循**“前肯后否，前否后肯”**的镜像反转法则：

### 1. 前肯后否
*   **例句**：The UI component is green, **isn't it**?
    *   **中文翻译**：这个 UI 组件是绿色的，对吧？
    *   **结构解析**：前半句为肯定句（is），尾部使用否定助动词提问（isn't it，其中 it 代指 component）。

### 2. 前否后肯
*   **例句**：The server is not ready, **is it**?
    *   **中文翻译**：服务器还没准备好，对吧？
    *   **结构解析**：前半句为否定句（is not），尾部使用肯定形式（is it）提问。
