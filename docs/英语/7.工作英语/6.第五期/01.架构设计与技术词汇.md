# 第五期 - 01. 架构设计与技术词汇

## 核心词汇精讲

### 1. proposal
*   **音标/词性**：`/prəˈpəʊzl/` (英) `/prəˈpoʊzl/` (美) · `n.`
*   **核心释义**：提案，提议，建议（在 IT 开发中常指“架构提案”、“设计方案”等方案性文档）
*   **词根解析**：pro-（向前）+ pos（放，来自拉丁语 ponere）+ -al（名词后缀）-> 向前摆出来的想法或计划 -> 提案，提议。
*   **常用搭配**：
    *   **architecture proposal** 架构设计方案
    *   **submit a proposal** 提交提案
    *   **project proposal** 项目建议书
*   **例句与拆解**：
    *   **例句**：I read the architecture **proposal** yesterday.
        *   **中文翻译**：我昨天阅读了架构设计方案。
        *   **结构解析**：I (主语) + read (谓语，过去式) + the architecture proposal (宾语) + [yesterday] (时间状语)。

### 2. interception
*   **音标/词性**：`/ˌɪntəˈsepʃn/` (英) `/ˌɪntərˈsepʃn/` (美) · `n.`
*   **核心释义**：拦截，监听，截获（在开发中指拦截器、中间件对请求或数据的拦截处理）
*   **词根解析**：inter-（在...之间）+ cept（抓，拿，来自拉丁语 capere）+ -ion（名词后缀）-> 在中途将其抓住 -> 拦截，截断。
*   **常用搭配**：
    *   **request interception** 请求拦截
    *   **packet interception** 数据包拦截
    *   **interception middleware** 拦截中间件
*   **例句与拆解**：
    *   **例句**：The gateway handles request **interception**.
        *   **中文翻译**：该网关处理请求拦截。
        *   **结构解析**：The gateway (主语) + handles (谓语，单三) + request interception (宾语)。

### 3. on-demand
*   **音标/词性**：`/ˌɒn dɪˈmɑːnd/` (英) `/ˌɑːn dɪˈmænd/` (美) · `adj. & adv.`
*   **核心释义**：按需的，随选的，即时的（常指按需加载、按需引入、按需计算等）
*   **词根解析**：on（按照）+ de-（加强）+ mand（命令/委托，来自拉丁语 mandare）-> 按照请求或命令触发的 -> 按需的。
*   **常用搭配**：
    *   **on-demand import** 按需引入/按需导入
    *   **on-demand loading** 按需加载
    *   **on-demand computing** 按需计算
*   **例句与拆解**：
    *   **例句**：Vue3 supports **on-demand** import.
        *   **中文翻译**：Vue3 支持按需引入。
        *   **结构解析**：Vue3 (主语) + supports (谓语，单三) + on-demand import (宾语)。

### 4. advantage
*   **音标/词性**：`/ədˈvɑːntɪdʒ/` (英) `/ədˈvæntɪdʒ/` (美) · `n.`
*   **核心释义**：优势，好处，有利条件
*   **词根解析**：ad-（向...，去...，在法语中演变为 av- 表示前进/在前面）+ vant（前面，来自拉丁语 abante）+ -age（名词/状态后缀）-> 处于前方的优势 -> 优势。
*   **常用搭配**：
    *   **have many advantages** 有很多优势
    *   **take advantage of** 利用...，占便宜
    *   **competitive advantage** 竞争优势
*   **例句与拆解**：
    *   **例句**：Tailwind CSS has many **advantages**.
        *   **中文翻译**：Tailwind CSS 有很多优势。
        *   **结构解析**：Tailwind CSS (主语) + has (谓语) + many advantages (宾语)。

### 5. maintainable
*   **音标/词性**：`/meɪnˈteɪnəbl/` · `adj.`
*   **核心释义**：可维护的，易保养的（在 IT 领域常指代码具备良好的可扩展性和可读性，便于后续修补）
*   **词根解析**：main（手，来自拉丁语 manus）+ tain（握住，保持，来自拉丁语 tenere）+ -able（能...的，形容词后缀）-> 能够一直握持住的 -> 可维护的。
*   **常用搭配**：
    *   **maintainable code** 可维护的代码
    *   **maintainable architecture** 易维护的架构
    *   **highly maintainable** 高度可维护的
*   **例句与拆解**：
    *   **例句**：Your TypeScript code is very **maintainable**.
        *   **中文翻译**：你的 TypeScript 代码非常可维护。
        *   **结构解析**：Your TypeScript code (主语) + is (系动词) + very (程度副词状语) + maintainable (形容词作表语)。

### 6. encapsulate
*   **音标/词性**：`/ɪnˈkæpsjuleɪt/` (英) `/ɪnˈkæpsəleɪt/` (美) · `v.`
*   **核心释义**：封装，包裹，简要概括（在面向对象编程中指将数据和方法隐藏在类内部）
*   **词根解析**：en-（进入...之中）+ capsule（小胶囊，来自拉丁语 capsa 箱子）+ -ate（动词后缀）-> 放入小胶囊中 -> 封装。
*   **常用搭配**：
    *   **encapsulate logic** 封装逻辑
    *   **encapsulate data** 封装数据
    *   **encapsulate details** 封装实现细节
*   **例句与拆解**：
    *   **例句**：This service **encapsulates** the API logic.
        *   **中文翻译**：此服务封装了 API 逻辑。
        *   **结构解析**：This service (主语) + encapsulates (谓语，单三) + the API logic (宾语)。

### 7. bitwise
*   **音标/词性**：`/ˈbɪtwaɪz/` · `adj. & adv.`
*   **核心释义**：按位（运算）的，位元的（指对二进制位进行的操作，如与、或、非、异或）
*   **词根解析**：bit（二进制位，由 binary digit 缩短而来）+ -wise（方向，方式，后缀）-> 按二进制位的方式 -> 按位的。
*   **常用搭配**：
    *   **bitwise operators** 按位运算符
    *   **bitwise operations** 位运算
    *   **bitwise shift** 按位位移
*   **例句与拆解**：
    *   **例句**：We use **bitwise** operators for flags.
        *   **中文翻译**：我们使用按位运算符来处理标志位。
        *   **结构解析**：We (主语) + use (谓语) + bitwise operators (宾语) + [for flags] (目的状语)。

### 8. anonymous
*   **音标/词性**：`/əˈnɒnɪməs/` (英) `/əˈnɑːnɪməs/` (美) · `adj.`
*   **核心释义**：匿名的，无名字的（在 IT 领域常指“匿名函数”、“匿名类”或“匿名访问”）
*   **词根解析**：an-（无，没有）+ onym（名字，来自希腊语 onyma）+ -ous（形容词后缀）-> 没有名字的 -> 匿名的。
*   **常用搭配**：
    *   **anonymous function** 匿名函数
    *   **anonymous class** 匿名类
    *   **anonymous user** 匿名用户
*   **例句与拆解**：
    *   **例句**：This is an **anonymous** function in Kotlin.
        *   **中文翻译**：这是 Kotlin 中的一个匿名函数。
        *   **结构解析**：This (主语) + is (系动词) + an anonymous function (表语) + [in Kotlin] (地点状语)。

### 9. decouple
*   **音标/词性**：`/ˌdiːˈkʌpl/` · `v.`
*   **核心释义**：解耦，分离，消除耦合关系（使系统的不同模块互不干扰）
*   **词根解析**：de-（反向，除去）+ couple（连接，结合，来自拉丁语 copula 纽带）-> 除去成对的连接 -> 解耦。
*   **常用搭配**：
    *   **decouple A from B** 将 A 与 B 解耦
    *   **decouple components** 组件解耦
    *   **loosely decoupled** 松散解耦的
*   **例句与拆解**：
    *   **例句**：We should **decouple** the frontend from the backend.
        *   **中文翻译**：我们应当将前端与后端解耦。
        *   **结构解析**：We (主语) + should (情态动词) + decouple (谓语动词原形) + the frontend (宾语) + [from the backend] (介词短语作状语/搭配，decouple A from B)。

### 10. canonical
*   **音标/词性**：`/kəˈnɒnɪkl/` (英) `/kəˈnɑːnɪkl/` (美) · `adj.`
*   **核心释义**：标准的，权威的，规范的，最简式的（在 SEO 中指“规范网页链接”，在数据格式中指“标准规范格式”）
*   **词根解析**：canon（标准，经典，源于希腊语 kanon 直杆/标准）+ -ical（形容词后缀）-> 符合标准的 -> 规范的。
*   **常用搭配**：
    *   **canonical URL** 规范网址/规范 URL
    *   **canonical data format** 标准数据格式
    *   **canonical path** 规范路径
*   **例句与拆解**：
    *   **例句**：Use the **canonical** URL for this page.
        *   **中文翻译**：使用此页面的规范 URL。
        *   **结构解析**：Use (祈使句谓语动词) + the canonical URL (宾语) + [for this page] (对象/目的状语)。

### 11. manipulation
*   **音标/词性**：`/məˌnɪpjuˈleɪʃn/` · `n.`
*   **核心释义**：操作，处理，控制，操纵（在前端通常指 DOM 操作，在后端指数据处理）
*   **词根解析**：mani-（手，来自拉丁语 manus）+ ple-（填充，抓满）+ -ation（名词后缀）-> 用手抓满并进行操作 -> 操作，操纵。
*   **常用搭配**：
    *   **DOM manipulation** DOM 操作
    *   **data manipulation** 数据处理/数据操作
    *   **string manipulation** 字符串处理
*   **例句与拆解**：
    *   **例句**：DOM **manipulation** can slow down the app.
        *   **中文翻译**：DOM 操作可能会降低应用的运行速度。
        *   **结构解析**：DOM manipulation (主语) + can (情态动词) + slow down (动词短语作谓语) + the app (宾语)。

### 12. offsetting
*   **音标/词性**：`/ˌɒfˈsetɪŋ/` (英) `/ˌɑːfˈsetɪŋ/` (美) · `n. & v. (offset的现在分词/动名词)`
*   **核心释义**：抵消，补偿，偏置（在 CSS 中指“边距偏置”，在算法中指“补偿计算”）
*   **词根解析**：off（偏离，离开）+ setting（放置）-> 偏置以消除或抵消负面影响 -> 抵消，偏置。
*   **常用搭配**：
    *   **offsetting margin** 偏置边距
    *   **offsetting effect** 抵消作用
    *   **offsetting errors** 抵消误差
*   **例句与拆解**：
    *   **例句**：**Offsetting** the margin fixes the layout.
        *   **中文翻译**：偏置边距修复了布局问题。
        *   **结构解析**：Offsetting the margin (动名词短语作主语) + fixes (谓语，单三) + the layout (宾语)。

### 13. snapshot
*   **音标/词性**：`/ˈsnæpʃɒt/` (英) `/ˈsnæpʃɑːt/` (美) · `n.`
*   **核心释义**：快照，简况（指在某一特定时刻对系统、数据库或状态的完整记录）
*   **词根解析**：snap（猛咬，快速拍照）+ shot（射击，照片）-> 快速拍下的画面 -> 快照。
*   **常用搭配**：
    *   **database snapshot** 数据库快照
    *   **state snapshot** 状态快照
    *   **take a snapshot** 创建快照
*   **例句与拆解**：
    *   **例句**：Take a **snapshot** of the database.
        *   **中文翻译**：对数据库进行一次快照备份。
        *   **结构解析**：Take (祈使句谓语动词) + a snapshot (宾语) + [of the database] (定语，修饰 snapshot)。

### 14. persist
*   **音标/词性**：`/pəˈsɪst/` (英) `/pərˈsɪst/` (美) · `v.`
*   **核心释义**：持久化，坚持，持续存在（在开发中指将内存数据写入持久化介质，如数据库或文件）
*   **词根解析**：per-（始终，彻底）+ sist（站立，来自拉丁语 sistere）-> 始终站立着 -> 持续存在，持久化。
*   **常用搭配**：
    *   **persist data** 持久化数据
    *   **persist state** 持久化状态
    *   **persist in doing something** 坚持做某事
*   **例句与拆解**：
    *   **例句**：We need to **persist** the user data.
        *   **中文翻译**：我们需要持久化用户数据。
        *   **结构解析**：We (主语) + need (谓语) + to persist the user data (动词不定式短语作宾语)。
